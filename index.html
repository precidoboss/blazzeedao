What is BlazDAO?

BlazDAO is a decentralised clipping economy built natively on top of Blaze.stream. Streamers post $BLAZE-denominated bounties tied to their real Blaze VODs and streams. Clippers watch the stream, capture the best moment, post the clip to X, and submit the link on-chain. The escrowed $BLAZE pays out automatically via smart contract — no middlemen, no manual payments, no trust required.

🎯 Problem it solves

Streamers want their streams clipped and shared to grow their audience. Clippers want to be compensated for quality content creation. Today both sides operate informally with no guarantee of payment. BlazDAO puts bounties on-chain and pays clippers in $BLAZE the moment a winner is chosen.

🔥 Why Blaze?

Blaze is the only web3-native streaming platform where creators earn on-chain and the audience is already crypto-native. $BLAZE as a reward currency is natural. We use Blaze OAuth for identity, Blaze channel APIs for live status and VOD data, and Blaze clips API for verification. BlazDAO is Blaze-first, not a Twitch clone bolted onto crypto.

How It Works

1. Sign Up

Connect your EVM wallet and link your Blaze.stream account via OAuth. Your Blaze identity is verified server-side — username, avatar, and channel stats pulled directly from the Blaze API. No NFT minting required.

2. Streamer Posts Bounty

Streamers lock $BLAZE in escrow via smart contract. The bounty is tied to a specific Blaze VOD (picked from real VOD history via the Blaze API). Two modes: Streamer Picks or Community Vote.

3. Clippers Submit

Clippers watch the Blaze stream, clip the best moment, post it to X, and submit the X link on-chain. The submission is recorded immutably on Avalanche. Clippers see real live status and follower counts powered by the Blaze API.

4. Winner Picked

Streamer reviews submissions and picks winner(s) with custom $BLAZE splits. In Community Vote mode, the contract resolves automatically once the deadline passes. Ties are handled on-chain.

5. Automatic Payout

Winning clipper(s) call claim() and receive their $BLAZE directly in their wallet. Pull-payment pattern — no manual payment, no platform fee, no trust required.

Blaze API Usage

BlazDAO integrates Blaze's developer API throughout the entire user journey — identity verification, live stream data, VOD history, clips, and channel stats. This is the core of our Blaze Builder Challenge entry.

Authentication & Identity

POST
blaze.stream/bapi/oauth2/generate-auth-url
Generates a PKCE auth URL for user login — the primary identity mechanism replacing wallet-only sign-up.
POST
blaze.stream/bapi/oauth2/token
Exchanges auth code + PKCE verifier for access + refresh tokens. Stored server-side in Supabase, never exposed to the frontend.
POST
blaze.stream/bapi/oauth2/refresh
Silently refreshes expired tokens in our backend. Auto-refresh means users never see an auth error mid-session.

Profile & Channel Data

GET
api.blaze.stream/v1/users/profile
Authenticated user profile — username, displayName, avatarUrl, user ID. Powers dashboard profile card, nav badge, and bounty card streamer info.
GET
api.blaze.stream/v1/channels/{channelId}/stats
Follower count, subscriber count, total views. Shown on dashboard overview and bounty cards as social proof.
GET
api.blaze.stream/v1/channels/{channelId}/live-stats
Real-time live status, viewer count, stream title. Powers the pulsing LIVE badge on bounty cards and live viewer count on overview.
GET
api.blaze.stream/v1/channels/{channelId}/vods
Recent VODs with titles, durations, thumbnails. Streamers pick which VOD their bounty targets — one click auto-fills the bounty form.
GET
api.blaze.stream/v1/channels/clips?channelId=
Recent Blaze clips — clipId, title, viewCount, previewImgUrl. Shown as channel activity. Planned: use clipId to verify a submitted X clip originated from this Blaze channel.
Smart Contracts

ClipDAOEscrow — Fuji Testnet

0xc515aaB3b3BBDC2313dc72e1603E0E1ccB419Ee1

Core escrow contract. Bounty creation, clip submission, voting, winner selection, pull-pattern payout. Built with ReentrancyGuard, Pausable, Ownable (OpenZeppelin v5). ERC-20 native ($BLAZE).

Mock $BLAZE Token — Fuji Testnet

0x57BAEAC484A0F3d5694d81420402FE54D4fBfec7

Standard ERC-20 (18 decimals) for testnet. Includes faucet() with 1-hour cooldown minting 10,000 mBLAZE per claim. Will be replaced with real $BLAZE on mainnet.

Contract Functions

postBounty()
Lock $BLAZE in escrow, set VOD ref, description, mode, deadline
submitClip()
Submit an X clip link on-chain tied to a bounty
voteSubmission()
Community vote — one vote per wallet per bounty
pickWinner()
Streamer picks winner(s) with custom $BLAZE splits
claim()
Pull-pattern: winner withdraws their $BLAZE reward
cancelBounty()
Cancel within 1hr grace period for full refund

Security Highlights

Pull-payment on all transfers prevents reentrancy. SafeERC20 guards against non-standard tokens. Splits must sum exactly to bounty reward. Streamer cannot submit/vote on their own bounty. One submission + one vote per wallet. 200-submission cap prevents gas-griefing. Admin can only sweep funds unclaimed for 180+ days.

Testnet to Mainnet Plan

Current state

BlazDAO is fully functional on Avalanche Fuji testnet. Real user flows work end-to-end: Blaze OAuth login, bounty creation, clip submission, winner selection, and $BLAZE payout — all on testnet with mock tokens. The Blaze API integration is production-ready.

How Hackathon Funding Gets Us to Mainnet

Marketing to bring streamers and clippers onto the platform, server infrastructure and storage costs (Vercel, Supabase, Render), tools and resources for scaling the backend, initial $BLAZE liquidity for the escrow contract, and community growth budget. We are applying to the Blaze Builder Challenge specifically because the prize provides exactly the bridge needed to move from a working testnet product to a live mainnet platform with real $BLAZE rewards for real clippers.
